# v0.6.4 Mobile Toolbar and Sidebar Parity Matrix

Milestone: [0.6.4](https://github.com/h3pdesign/Neon-Vision-Editor/milestone/16)

Use this matrix for issues [#91](https://github.com/h3pdesign/Neon-Vision-Editor/issues/91)
and [#92](https://github.com/h3pdesign/Neon-Vision-Editor/issues/92). The goal is
to keep compact iPhone and iPad layouts discoverable without adding IDE-style UI.

## Toolbar Actions

| Action | Symbol | macOS | iPhone | iPad | Accessibility expectation |
|---|---|---|---|---|---|
| Open File | `doc.badge.plus` / platform import symbol | Toolbar/menu opens importer | Scrollable toolbar or overflow opens importer | Toolbar opens importer | Button label names the action and importer sheet receives focus |
| Open Folder | `folder` | Toolbar/menu opens folder picker | Sidebar action opens folder picker where supported | Toolbar/sidebar opens folder picker | Folder action is not silent; disabled state is announced when unavailable |
| New File | `doc.badge.plus` | Creates a new untitled tab or prompts as configured | Filename dialog remains visible until submit/cancel | Same as iPhone plus keyboard submit | Dialog has a clear title, text field label, cancel, and create button |
| New Folder | `folder.badge.plus` | Prompts for folder name | Prompts for folder name from project sidebar | Same as iPhone | Focus starts in the name field and returns to sidebar after dismissal |
| Quick Open | `magnifyingglass` | Opens quick-open UI | Opens compact quick-open UI | Opens keyboard-friendly quick-open UI | Search field is first in focus order |
| Settings | `gearshape` | Opens Settings window | Opens Settings sheet | Opens Settings sheet | Settings remains reachable outside overflow on iPad |
| Toolbar Help | `questionmark.circle` | Opens Help from toolbar/menu/settings | Opens Help from toolbar or overflow | Must appear directly in toolbar, not only the 3-dot menu | Help icon has a readable label and no clipped text |
| Markdown Preview | `doc.richtext` | Opens preview pane | Opens preview sheet where available | Opens preview pane/sheet | Preview state is announced and focus can return to editor |
| PDF Mode | `doc.text` | Chooses Paginated Fit or One Page Fit | Chooses mode without clipping labels | Chooses mode without clipping labels | Current mode is exposed through selected state |
| Export PDF | `square.and.arrow.down` | Saves PDF | Opens document exporter | Opens document exporter | Export progress and errors remain announced |
| Code Snapshot | `camera.viewfinder` | Opens snapshot controls | Defaults to wrap mode | Defaults to chosen editor behavior | Wrap setting is visible and keyboard accessible |
| Diff View | `arrow.left.arrow.right` | Compare local vs disk or tab vs tab | Header text remains compact and readable | Header text remains readable in split layouts | Previous/Next Change buttons keep labels and disabled traits |

## Project Sidebar Actions

| Sidebar action | iPhone expected result | iPad expected result | Regression risk |
|---|---|---|---|
| Tap Open File | Presents file importer dialog | Presents file importer dialog | Silent tap or dismissed dialog |
| Tap Open Folder | Presents folder importer dialog where supported | Presents folder importer dialog | Action only works from overflow |
| Tap New File | Filename dialog stays open until explicit action, then opens a new tab | Same, with hardware keyboard submit | Dialog disappears immediately or no tab is created |
| Tap New Folder | Folder-name dialog stays open until explicit action | Same, with tree refresh | Sidebar refresh misses new folder |
| Tap Help | Opens Toolbar Help | Opens Toolbar Help directly from toolbar | Help is only reachable in 3-dot menu |

## Compact Layout Checks

- iPhone portrait: toolbar can scroll horizontally, symbols remain tappable, and labels do not wrap into clipped controls.
- iPhone landscape: diff header title, previous/next controls, and file chips fit without excessive line breaks.
- iPad portrait split view: Markdown preview text size remains readable and PDF mode controls are not clipped.
- iPad landscape with keyboard: tab order moves through toolbar, sidebar actions, editor, preview, and dialogs without trapping focus.
